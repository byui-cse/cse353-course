About OpenCV

OpenCV (Open Source Computer Vision Library) is an open source computer vision and machine learning software library. OpenCV was built to provide a common infrastructure for computer vision applications and to accelerate the use of machine perception in the commercial products. Being a BSD-licensed product, OpenCV makes it easy for businesses to utilize and modify the code.

The library has more than 2500 optimized algorithms, which includes a comprehensive set of both classic and state-of-the-art computer vision and machine learning algorithms. These algorithms can be used to detect and recognize faces, identify objects, classify human actions in videos, track camera movements, track moving objects, extract 3D models of objects, produce 3D point clouds from stereo cameras, stitch images together to produce a high resolution image of an entire scene, find similar images from an image database, remove red eyes from images taken using flash, follow eye movements, recognize scenery and establish markers to overlay it with augmented reality, etc. OpenCV has more than 47 thousand people of user community and estimated number of downloads exceeding 14 million. The library is used extensively in companies, research groups and by governmental bodies.

Along with well-established companies like Google, Yahoo, Microsoft, Intel, IBM, Sony, Honda, Toyota that employ the library, there are many startups such as Applied Minds, VideoSurf, and Zeitera, that make extensive use of OpenCV. OpenCV’s deployed uses span the range from stitching streetview images together, detecting intrusions in surveillance video in Israel, monitoring mine equipment in China, helping robots navigate and pick up objects at Willow Garage, detection of swimming pool drowning accidents in Europe, running interactive art in Spain and New York, checking runways for debris in Turkey, inspecting labels on products in factories around the world on to rapid face detection in Japan.

It has C++, C, Python, Java and MATLAB interfaces and supports Windows, Linux, Android and Mac OS. OpenCV leans mostly towards real-time vision applications and takes advantage of MMX and SSE instructions when available. Using the shape attribute¶

Image loaded into OpenCV will have 3 values (ie., (300, 400, 3))

  • first value is the number of rows
  • second value is the number of columns
  • third value is the number of channels

A channel contains values for a color. In a RGB (Red, Green Blue) image, there are 3 channels. An 8-bit image would have 1 channels. Values in channels are from 0 to 255. They are of data type np.uint8.

In [5]:
# Read the image file and display it's contents
print('The shape of the image =', image.shape)

The shape of the image = (300, 300, 3)

Dividing channels from an image¶

You use the function OpenCV split() feature to divide channels. Note, that split() returns Blue, Green, Red in that order.

In [6]:
blue, green, red = cv2.split(image)
print('shape of the blue channel =', blue.shape)
print('shape of the green channel =', green.shape)
print('shape of the red channel =', red.shape)

shape of the blue channel = (300, 300)
shape of the green channel = (300, 300)
shape of the red channel = (300, 300)

Merging channels to create an image¶

You use the function OpenCV merge() feature to divide channels. Note that merge() takes 1 tuple of the different channels.

In [7]:
new_image = cv2.merge((blue, green, red))
print('Shape of new image =', new_image.shape)

# Display image in this document
plt.imshow(cv2.cvtColor(new_image, cv2.COLOR_BGR2RGB))
plt.show()

Shape of new image = (300, 300, 3)
In [8]:
# Creating a grayscale version of an image from the blue channel
new_image = cv2.merge((blue, blue, blue))
print('Shape of new image =', new_image.shape)

# Display image in this document
plt.imshow(cv2.cvtColor(new_image, cv2.COLOR_BGR2RGB))
plt.show()

Shape of new image = (300, 300, 3)

Modifing Images¶

This following section will use numpy to change image pixels. This is a warning from the OpenCV website:

Numpy is an optimized library for fast array calculations. So simply accessing each and every pixel value and modifying it will be very slow and it is discouraged.

Reading a pixel¶

Since an image in OpenCV is a numpy array, we can use [] to access pixels

In [9]:
rows, cols, _ = image.shape
print(f'Image has {rows} rows and {cols} columns')

pixel = image[180, 100]
print('Pixel at (180, 100) =', pixel)
print('The type of the pixel is =', type(pixel))

Image has 300 rows and 300 columns
Pixel at (180, 100) = [ 84  85 205]
The type of the pixel is = <class 'numpy.ndarray'>

Changing a pixel¶

In [10]:
# Change pixel [180, 100]
# Note that something like "image[180, 100]" returns a NEW numpy array
pixel = image[180, 100]
print('Orginial Pixel at (180, 100) =', pixel)
pixel = [255, 255, 255]
print('Changed Pixel at (180, 100) =', pixel)
print('Pixel at image[180, 100] =', image[180, 100])

Orginial Pixel at (180, 100) = [ 84  85 205]
Changed Pixel at (180, 100) = [255, 255, 255]
Pixel at image[180, 100] = [ 84  85 205]

Changing multiple pixels¶

You can loop through a numpy array to change values, however, there is an easier method

In [12]:
# Original Image
print('Original Image')
plt.imshow(cv2.cvtColor(image, cv2.COLOR_BGR2RGB))
plt.show()

# Copy the image for changes.  "new_image = image" will NOT work
new_image = image.copy()

print('new image')

# Change a small square 50 x 50 that starts a pixel [21, 34] to the color blue
# OpenCV are [Blue, Green, Red]
new_image[21:72, 34:85] = [255, 0, 0]

plt.imshow(cv2.cvtColor(new_image, cv2.COLOR_BGR2RGB))
plt.show()

Saving images¶

Saving images is easy with the function imwrite().

In [13]:
# Read the image file and display it's contents
image = cv2.imread('tricycle.png')

# save image with filename 'temp.png'
cv2.imwrite('temp.png', image)

# Read the image file and display it's contents
image2 = cv2.imread('temp.png')

# Display image in this document
plt.imshow(cv2.cvtColor(image2, cv2.COLOR_BGR2RGB))
plt.show()

Creating images¶

Of course, you can create images using numpy. It is very important to create the numpy array with the correct data type. Images in OpenCV are of datatype uint8

In [14]:
# Create an 8-bit 200 x 300 image
image = np.zeros((200, 300), dtype=np.uint8)
print('Shape =', image.shape)

Shape = (200, 300)
In [15]:
# Create an 24-bit 200 x 300 image
image = np.zeros((200, 300, 3), dtype=np.uint8)
print('Shape =', image.shape)

Shape = (200, 300, 3)
